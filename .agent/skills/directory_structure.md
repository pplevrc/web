# Directory Structure

The project's directory structure and the responsibilities of each directory are as follows:

```text
public/                                    # Static assets (expected to be copied as-is during build)
public-mock/                               # Mock data (expected to be copied during build)
scripts/                                   # Scripts referenced during the build process
src/
├── assets/                                # Static assets (expected to be referenced by Vite)
├── pages/
│   └── (path-of-page).astro               # Responsible ONLY for routing, layout, and data fetching
├── components/
│   ├── pages/                             # Root directory for page-specific UI components
│   │   └── (name-of-page)/
│   │       ├── index.astro                # UI entry point for the page
│   │       └── (component-name)/
│   │           └── (component-name).astro # Page-specific components
│   └── (domain-name)/                     # Generic or domain-specific common components
│       └── (component-name)/
│           └── (component-name).astro     # Generic or domain-specific common components
├── contents/
│   └── (content-name)/
│       └── index.ts                       # Logic for fetching content from CMS
├── layouts/
│   └── (layout-name).astro                # Layout components
├── styles/                                # Stylesheets (auto-generated by PandaCSS)
└── lib/
    └── (library-name).ts                  # Generic logic and libraries
```
